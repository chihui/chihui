Github 使用总结

仓库

假设本地有一个通过git init 创建出来的仓库

需求：将这个本地仓库放到线上远程仓库。

1.在第三方托管服务那里新建一个空仓库
2.拿到空仓库的地址
3.使用git remote add abc git@github.com:chihui/bbb.git
4.使用git remote show看是否添加上（如果想放在其他第三方托管服务上，可以直接利用add添加的方式，可多个进行托管）
5.使用git push abc master添加到线上
6.通过上面的操作之后，以后每次去push的时候都必须显示的写上git push 远端仓库地址别名 分支名称

如果不想每次push的时候都写那么长，（没有多个远程仓库，也不需要频繁地去push不同的分支，假设每次都是push到abc master,
那么可以通过git push --set-upstream abc master去推送一次，那么在以后的push中,只要仓库地址没变，分支不变，就可以直接git push）

本地没有，线上有

需求：想要获取远程仓库到本地

1.使用 git clone 远程仓库地址

只要执行了上面的操作之后，git会自动帮你通过网络去下载对应的远程仓库，下载下来之后，自动添加一个remote
,地址是你 clone的那个地址，名字叫 origin，之后，只要你想要push，直接git push origin master就可以了，
原因是它已经自动帮你建立了上下游关系。

注：提交的时候，如果出现403 ，说明自己没有权限，需要找管理仓库的工作人员，开通权限。
想要删除文件，但是又不知道哪个程序占用，删除不了，可以使用命令rm -rf 文件名/进行删除

多人协作

假设现在线上最新版本是 : ….(打开Commits就能看到)

找到文件内容：

push

添加仓库权限，可点击Settings，然后点击Collaborators 进行邀请，点击add Collaborators进行邀请添加
收取邮件，同意邀请

权限文件

对于这个权限，首先你需要在你公司所使用的第三方托管服务那里注册一个账号，
然后告诉你的源代码仓库管理员你的账号是多少，他会给你配置项目的访问提交权限，
他给你配置之后，你需要去你的当时注册的邮箱中收一封邮件，然后同意参与该项目开发，
然后你才有该远程仓库项目协同权限。

1.不同的用户分别去线上克隆
2.如果其中一个a改动提交，另外一个b需要pull去拿到最新的代码，进行解决冲突

冲突：假设b也进行改动提交，同时a又进行改动提交，这时，线上最新版本是b提交的，和本地的最新版本冲突了，
会报错，所以，要把线上的最新版本拿下来，把对方写的代码喊过来，商量一下！
